# ADR-0002: 헥사고날 모듈러 모놀리스

- 상태: 확정 (2026-07-17)
- 맥락: "모놀리스로 가되 관심사가 다르면 분리" 요구. MSA는 진짜 필요할 때만.

## 결정
단일 배포 **모놀리스** + Gradle 멀티모듈(헥사고날). 모듈 경계 = 미래의 네트워크 경계.

```
app → (chat-api, ws-gateway) → chat-core ← (storage-dynamo, fanout, async-sqs, integration)
```

## 근거
- `chat-core`를 프레임워크·AWS 무의존으로 두면 도메인이 인프라 교체에 흔들리지 않는다.
- 어댑터를 모듈로 격리하면 **분리해야 할 때 그 경계가 그대로 배포 경계**가 된다. 특히 `ws-gateway`(연결 밀도·배포 주기가 다름)를 최우선 분리 후보로 격리.

## 규칙
- 어댑터 모듈끼리 직접 참조 금지. 교차 필요 시 `chat-core`에 port 추가.
- 관심사 분리는 **패키지 먼저**. 새 모듈은 독립 배포·빌드 격리·팀 경계 중 하나가 실제로 생길 때만.

## 대가
- 지금은 실질 로직이 `chat-core`뿐이라 8모듈이 다소 과함. 단 `ws-gateway` Netty 분리 계획이 이를 정당화(안 하면 단일/2모듈로 축소 검토). 관련 논의는 대화 이력 참조.
